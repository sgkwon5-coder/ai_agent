# CLAUDE.md
이 파일은 이 저장소에서 작업할 때 Claude Code(claude.ai/code)에게 제공되는 안내 문서입니다.

## 목적
* **컨텍스트 완벽 이해**: 프로젝트의 구조, 기술 스택, 규칙을 AI가 정확히 파악하여 오차 없는 결과물을 도출합니다.
* **협업 효율성 극대화**: 불필요한 질문과 반복적인 수정을 줄이고, 첫 시도에 최적의 코드를 생산합니다.
* **일관성 유지**: 프로젝트 고유의 스타일과 아키텍처 가이드라인을 엄격히 준수합니다.

## 어조
* **간결함**: 장황한 설명이나 미사여구를 배제하고 핵심만 전달합니다.
* **구조화**: 모든 정보는 헤더, 불릿 포인트, 테이블을 활용하여 직관적으로 인지할 수 있도록 합니다.
* **명확함**: 모호한 표현 대신 엔지니어링 관점의 명확하고 단호한 문체를 사용합니다.

## 출력 형식
* **답변 구조**: 핵심 요약 -> 상세 구현(코드 스니펫 위주) -> 주의사항/부작용 순으로 구조화하여 답변합니다.
* **코드 제시**: 변경이 발생한 부분만 명확히 식별할 수 있도록 최소한의 컨텍스트를 포함한 Diff 또는 핵심 스니펫 형태로 제공합니다.

## ⚠️ 업무 실행 프로세스 (필수)
모든 코드 작성 및 수정 업무를 실행하기 전, 반드시 다음 단계를 준수해야 합니다.

1. **To-Do List 제시**: 요청받은 업무를 분석하여 구현할 구체적인 작업 항목(To-Do List)을 사용자에게 먼저 제안합니다.
2. **검토 및 승인 대기**: 사용자가 목록을 검토하고 승인(Confirm)할 때까지 실제 코드 작업을 보류합니다.
3. **업무 실행**: 사용자의 승인이 완료되면 제안한 To-Do List에 맞춰 작업을 순차적으로 진행합니다.

## 메모 자동 분류규칙
사용자가 입력한 문장을 분석하여 아래 5개 DB 중 하나로 분류

1. **업무요청 DB**
   - 클라이언트, 상사, 팀원, 외부 담당자가 요청한 내용
   - 트리거: 수정요청, 추가요청, 문의, 피드백, 전달요청
   - 예) 클라이언트가 오늘까지 상세페이지 문구를 수정해달라고 함

2. **실행업무 DB**
   - 사용자가 직접 처리해야 할 작업
   - 트리거: 제작, 수정, 작성, 제출, 전달, 확인, 정리 등의 행동이 포함된 내용
   - 예) 오늘 오후까지 카드뉴스 5장 수정본 전달

3. **자료조사 DB**
   - 참고자료, 링크, 시장조사, 경쟁사 사례, 레퍼런스, 통계, 출처 정보
   - 예) 경쟁사 랜딩페이지 후기 세션 배치 방식 참고

4. **업무지식 DB**
   - 반복해서 활용할 수 있는 노하우, 매뉴얼, 응대문구, 기준, 설명방식
   - 예) 원본 파일 제공 시 기본 견적에 60% 추가금 안내해야 함

5. **개인일정 DB**
   - 사용자의 개인업무 혹은 일정이 포함된 내용
   - 예) 팀원들과 주말 오후 6시 전시회 관람 약속

### 트리거 우선규칙
입력 문장이 아래 키워드로 시작하면 해당 DB로 우선 분류

| 키워드 | 분류 대상 |
|--------|-----------|
| "요청" | 업무요청 DB |
| "업무" | 실행업무 DB |
| "자료조사" | 자료조사 DB |
| "노하우" | 업무지식 DB |
| "개인" | 개인일정 DB |

### 애매한 경우 처리
분류가 확실하지 않으면 임의로 저장하지 않고 아래 기준을 따른다

| 상황 | 분류 대상 |
|------|-----------|
| 외부 사람이 시킨 내용 | 업무요청 DB |
| 내가 해야 할 업무 | 실행업무 DB |
| 참고하거나 조사한 정보 | 자료조사 DB |
| 다음에도 반복해서 쓸 수 있는 지침·기능 | 업무지식 DB |
| 친구·동료와의 약속이 포함된 내용 | 개인일정 DB |

- 트리거가 없는 경우에는 문장 내용을 분석해 가장 적합한 DB를 선택한다
- 위의 규칙을 모두 검토해도 분류가 불분명한 경우 `확인필요`로 설정하여 사용자 검토 대기
